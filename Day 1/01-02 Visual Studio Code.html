1. Start Lab Environment

You need to run following steps if your lab environment has shut down. To conserve resources, the lab environment shuts down automatically after eight hours.

1.1. Start Environment After Shut Down

  1. Go to the OPENTLC lab portal and use your OPENTLC credentials to log in.

  2. Navigate to Services → My Services (this should be the screen shown right after logging in).

  3. In the list of your services, select your lab environment.

  4. Select App Control → Start to start your lab environment.

  5. Select Yes at the Are you sure? prompt.

  6. On the bottom right, click Submit.

    After a few minutes, expect to receive an email letting you know that the lab environment has been started.

1.2. Connect bastion Node

  1. Bastion node is the main host which is accssible via ssh from public network. You will be using this host for running commands and accessing other host available in your environment. You will use ssh to connect bastion node, using the login details provided in third email.

    [laptop]$ ssh <OPENTLC-USER-ID>@bastion.${GUID}.${SUBDOMAIN}

  2. Switch to devops user. The User devops is service account in bastion node who has complete sudoer privilege to run all commands with no sudo password.

    [bastion]$ sudo su - devops

1.3. Refresh Satellite Manifest

  1. After restart of your classroom environment need to run following Lab Manager command to refresh satellite manifest.

    $ lab refresh -i satellite

2. Overview

In this lab you will install Visual Studio Code IDE and configure its settings and extensions on bastion node. It will be accssible from browser using classroom haproxy and you will be able to execute commands using it’s virtual termnial.

You need to perform all of the lab instruction on bastion node as devops user.

3. Introduction

Visual Studio Code IDE

Visual Studio Code is a source-code editor made by Microsoft for Windows, Linux and macOS. Features include support for debugging, syntax highlighting, intelligent code completion, snippets, code refactoring, and embedded Git.

3.1. Install VScode

  1. In this section of the lab you will install Visual Studio Code version 4.0.2 as follows.

    $ mkdir $HOME/vscode-ide
$ cd $HOME/vscode-ide
$ curl -fOL https://github.com/cdr/code-server/releases/download/v4.0.2/code-server-4.0.2-amd64.rpm
$ sudo dnf install code-server-4.0.2-amd64.rpm
$ sudo systemctl enable --now code-server@$USER

3.2. Setup Auth and Network

  1. In this section of lab you will configure authentication password and bind Visual Studio Coder server to run on bastion.example.com:8080.

    $ export BASTION_PRIVATE_IPADDR=$(ip a s eth0 | grep inet | head -n  1 | awk '{print $2}'|awk -F "/" '{print $1}')

$ export BASTION_PASSWORD='r3dh4t1!'

$ cat > $HOME/.config/code-server/config.yaml <<EOF
bind-addr: ${BASTION_PRIVATE_IPADDR}:8080
auth: password
password: ${BASTION_PASSWORD}
cert: false
EOF

3.3. Configure Settings

  1. You will configure following settings for the Visual studio Code as follows-

    $ mkdir -p $HOME/.local/share/code-server/User/
$ cat > $HOME/.local/share/code-server/User/settings.json <<EOF
{
    "git.ignoreLegacyWarning": true,
    "terminal.integrated.experimentalRefreshOnResume": true,
    "window.menuBarVisibility": "visible",
    "git.enableSmartCommit": true,
    "workbench.tips.enabled": false,
    "workbench.startupEditor": "readme",
    "workbench.colorTheme": "Default Dark+"
    "telemetry.enableTelemetry": false,
    "search.smartCase": true,
    "git.confirmSync": false
}
EOF

3.4. Install Extenions

  1. You need to download following Visual Studio Code extension.

    $ mkdir -p $HOME/.local/share/code-server/extensions
$ cd $HOME/.local/share/code-server/extensions/
$ export VSCODE_URL=https://gpte-public.s3.amazonaws.com/vscode-plugins
$ wget ${VSCODE_URL}/bierner.markdown-preview-github-styles-0.1.6.vsix
$ wget ${VSCODE_URL}/hnw.vscode-auto-open-markdown-preview-0.0.4.vsix
$ wget ${VSCODE_URL}/vscoss.vscode-ansible-0.5.2.vsix

  2. You can install Visual Studio Code extensions as follows-

    $ code-server --install-extension bierner.markdown-preview-github-styles-0.1.6.vsix
$ code-server --install-extension hnw.vscode-auto-open-markdown-preview-0.0.4.vsix
$ code-server --install-extension vscoss.vscode-ansible-0.5.2.vsix

3.5. Start Service

  1. You need to restart Visual Studio Code service to apply all the changes which you have made in previous steps.

    $ sudo systemctl restart code-server@$USER

3.6. HAproxy Configuration

  1. For your convenience we have already configured haproxy server for Visual Studio Code. You may need to run following Lab Manager command to restart haproxy server pod as follows.

    $ lab reconfigure -p haproxy
