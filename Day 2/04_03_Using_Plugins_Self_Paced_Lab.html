1. Start Lab Environment

You need to run following steps if your lab environment has shut down. To conserve resources, the lab environment shuts down automatically after eight hours.

1.1. Start Environment After Shut Down

  1. Go to the OPENTLC lab portal and use your OPENTLC credentials to log in.

  2. Navigate to Services → My Services (this should be the screen shown right after logging in).

  3. In the list of your services, select your lab environment.

  4. Select App Control → Start to start your lab environment.

  5. Select Yes at the Are you sure? prompt.

  6. On the bottom right, click Submit.

    After a few minutes, expect to receive an email letting you know that the lab environment has been started.

1.2. Connect bastion Node

  1. Bastion node is the main host which is accssible via ssh from public network. You will be using this host for running commands and accessing other host available in your environment. You will use ssh to connect bastion node, using the login details provided in third email.

    [laptop]$ ssh <OPENTLC-USER-ID>@bastion.${GUID}.${SUBDOMAIN}

  2. Switch to devops user. The User devops is service account in bastion node who has complete sudoer privilege to run all commands with no sudo password.

    [bastion]$ sudo su - devops

1.3. Refresh Satellite Manifest

  1. After restart of your classroom environment need to run following Lab Manager command to refresh satellite manifest.

    $ lab refresh -i satellite

  • Collection

    1. community.general

    2. ansible.posix

You need to perform all of the lab instruction on bastion node as devops user.

3. Adding Plugins Collections

In this lab you will be using $HOME/ee-osp-build directory which you had created and configured to build ee-osp custom execution environment in the previous lab. So you don’t need to re do everything to build image from scratch.

3.1. Add Galaxy Dependencies

  1. You just need to add following plugins collections in the galaxy requirements.yml file-

    • community.general

    • ansible.posix

      $ cd $HOME/ee-osp-build
$ cat requirements.yml
collections:
  - openstack.cloud
  - community.general (1)
  - ansible.posix (2)

      Callouts

      1Add community.general collection in the requirements.yml
      2Add ansible.posix collection in the requirements.yml

3.2. Re Build Execution Environment

  1. After adding collections in reuqirements.yml, You just need to re run ansible-builder command as follows-

    $ ansible-builder build -t ee-osp:1.1.0
    Build output
    File context/_build/requirements.yml had modifications and will be rewritten
Running command:
  podman build -f context/Containerfile -t ee-osp:1.1.0 context
Complete! The build context can be found at: /home/devops/ee-osp-build/context

3.3. Verify Execution Environment

  1. Let’s verify custom execution environment image.

    $ ansible-navigator images
    Example 1. Steps to verify collections

    1. Type the line number of ee-osp image to get details.

    2. Then type 2 to get list of collections, Make sure you see all of the collection, which you have installed, in the list.

    3. To exit, keep on pressing <ESC> key until you are back to shell prompt.

4. Using Plugins

In this part of the lab you will use callback plugins which was installed in the image from the collections. To use plugins you will create ansible.cfg, ansible-navigator.yaml and simple ansible playbook.

4.1. Create ansible.cfg

  1. You can create ansible.cfg to use callback plugins as follows-

    $ mkdir $HOME/test-plugins
$ cd $HOME/test-plugins
$ cat > ansible.cfg <<EOF
[defaults]
callbacks_enabled = ansible.posix.timer, ansible.posix.profile_tasks
EOF

4.2. Create Playbook

  1. You can create simple playbook to test plugins as follows-

    $ cat > playbook.yml <<EOF
---
- name: Test plugins
  hosts: localhost
  gather_facts: false
  vars:
    guid: "{{ lookup('env', 'GUID')}}"
  tasks:
    - name: Collect OpenStack instances info
      openstack.cloud.server_info:
        cloud: "{{ guid }}-project"
        server: "*"
EOF

4.3. Create Settings file

  1. You can prepare ansible-navigator settings file to use ansible.cfg and playbook as follows.

    $ cat > ansible-navigator.yaml <<EOF
ansible-navigator:
  ansible:
    config: ansible.cfg
    playbook: playbook.yml
  app: run
  execution-environment:
    image: ee-osp:1.1.0
    pull-policy: missing
    volume-mounts:
      - src: "/etc/openstack/clouds.yaml"
        dest: "/etc/openstack/clouds.yaml"
        label: "Z"
    environment-variables:
      pass:
        - GUID
  mode: stdout
  playbook-artifact:
    enable: False
EOF

4.4. Run Playbook

  1. To run your playbook you simply execute following command and verify the output.

    $ ansible-navigator
    Command Output

    Watch the output closely, This time replay has more infomation which is because of callbacks plugins.
