1. Start Lab Environment

You need to run following steps if your lab environment has shut down. To conserve resources, the lab environment shuts down automatically after eight hours.

1.1. Start Environment After Shut Down

  1. Go to the OPENTLC lab portal and use your OPENTLC credentials to log in.

  2. Navigate to Services → My Services (this should be the screen shown right after logging in).

  3. In the list of your services, select your lab environment.

  4. Select App Control → Start to start your lab environment.

  5. Select Yes at the Are you sure? prompt.

  6. On the bottom right, click Submit.

    After a few minutes, expect to receive an email letting you know that the lab environment has been started.

1.2. Connect bastion Node

  1. Bastion node is the main host which is accssible via ssh from public network. You will be using this host for running commands and accessing other host available in your environment. You will use ssh to connect bastion node, using the login details provided in third email.

    [laptop]$ ssh <OPENTLC-USER-ID>@bastion.${GUID}.${SUBDOMAIN}

  2. Switch to devops user. The User devops is service account in bastion node who has complete sudoer privilege to run all commands with no sudo password.

    [bastion]$ sudo su - devops

1.3. Refresh Satellite Manifest

  1. After restart of your classroom environment need to run following Lab Manager command to refresh satellite manifest.

    $ lab refresh -i satellite

3. Deployment Playbook

In this section you will write a playbook to deploy Three Tier Application on the above nodes. You will be all necessary roles and playbook from mycollections.threetier collection (created in earlier lab).

3.1. Create Inventory

  1. You have already inventory file in the repository so you just need to replace ssh key name as follows.

    $ cd $HOME/three-tier-app-roles
$ sed -i "s/\${GUID}/$GUID/g" inventory

3.2. Create Playbook

  1. You can create Three Tier Application deployment playbook inside the collection as follows:

    $ cd $HOME/three-tier-app-roles/collections/ansible_collections/mycollections/threetierapp/
$ mkdir playbooks
$ cd playbooks
$ cat > deploy.yml <<EOF
---
- name: Deploy database
  hosts: database_servers
  roles:
    - name: Install and configure firewall
      role: mycollections.threetierapp.firewall
      vars:
        firewall_services:
          - mysql
    - name: database deployment
      role: mycollections.threetierapp.mysqldb

- name: Deploy haproxy
  hosts: loadbalancers
  roles:
    - name: Install and configure firewall
      role: mycollections.threetierapp.firewall
      vars:
        firewall_services:
          - http
        firewall_tcp_ports:
          - 5000
    - name: frontend deployment
      role: mycollections.threetierapp.haproxy

- name: Deploy apache
  hosts: backend_servers
  roles:
    - name: Install and configure firewall
      role: mycollections.threetierapp.firewall
      vars:
        firewall_services:
          - http
        firewall_tcp_ports:
          - 8080
    - name: application deployment
      role: mycollections.threetierapp.apache
EOF

  2. Create main.yml playbook which will import playbook from collection as follows.

    $ cd $HOME/three-tier-app-roles
$ cat > main.yml <<EOF
---
- import_playbook: mycollections.threetierapp.deploy
EOF

3.3. Ansible Navigator Settings

  1. You can create Ansible Navigator settings file with default name ansible-navigator.yaml as follows.

    $ cat > ansible-navigator.yaml <<EOF
ansible-navigator:
  ansible:
    playbook: main.yml
    inventories:
      - inventory
  app: run
  execution-environment:
    image: ee-osp:1.2.0
    pull-policy: missing
  mode: stdout
  logging:
    append: False
    level: critical
    file: /tmp/threetierapp/ansible-navigator.log
  playbook-artifact:
    enable: True
    replay: /tmp/threetierapp/artifact.json
    save-as: /tmp/threetierapp/artifact.json
EOF

3.4. Push Content to Remote Repository

  1. Push all your work from local git repository to remote gitlab repository as follows.

    $ git add .
$ git commit -m "Re-useable Ansible Roles"
$ git push origin refactor-roles

5. Cleanup

You must need to destroy threetier nodes, before you move to other labs. Or, Reset in case, you want to redo the lab.

Destroy Lab Nodes

  1. If you want to move to next lab then please run following Lab Manager command to destroy lab nodes.

    $ lab destroy -i threetier
Reset Lab Nodes

  1. You can reset lab nodes if you want to re-do the lab.

    $ lab reset -i threetier
