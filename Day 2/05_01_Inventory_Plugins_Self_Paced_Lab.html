1. Start Lab Environment

You need to run following steps if your lab environment has shut down. To conserve resources, the lab environment shuts down automatically after eight hours.

1.1. Start Environment After Shut Down

  1. Go to the OPENTLC lab portal and use your OPENTLC credentials to log in.

  2. Navigate to Services → My Services (this should be the screen shown right after logging in).

  3. In the list of your services, select your lab environment.

  4. Select App Control → Start to start your lab environment.

  5. Select Yes at the Are you sure? prompt.

  6. On the bottom right, click Submit.

    After a few minutes, expect to receive an email letting you know that the lab environment has been started.

1.2. Connect bastion Node

  1. Bastion node is the main host which is accssible via ssh from public network. You will be using this host for running commands and accessing other host available in your environment. You will use ssh to connect bastion node, using the login details provided in third email.

    [laptop]$ ssh <OPENTLC-USER-ID>@bastion.${GUID}.${SUBDOMAIN}

  2. Switch to devops user. The User devops is service account in bastion node who has complete sudoer privilege to run all commands with no sudo password.

    [bastion]$ sudo su - devops

1.3. Refresh Satellite Manifest

  1. After restart of your classroom environment need to run following Lab Manager command to refresh satellite manifest.

    $ lab refresh -i satellite

3. Dynamic Inventory Plugins

Ansible requires an inventory to execute playbook tasks. Inventory can be prepared manually by creating static file and adding hosts. Or, It can be dynamic inventory plugins which pulls inventory information from other systems like satellite, cmddb, and etc.

3.1. List Instances

  1. To get any information from OpenStack cloud you must have cloud credentials. Cloud credentials are already stored in /etc/openstack/clouds.yaml file in your bastion node. So Let’s try to get list of servers/vms/instances from OpenStack cloud using openstack command line tool as follows.

    $ openstack server list -c Name -c Status -c Networks  --os-cloud=${GUID}-project
    Servers List

3.2. Build Inventory

You have seen bunch of servers in the output of openstack command. Now we need to build dynamic inventory using inventory plugins so that we can run automation playbooks on them.

  1. Let’s create dynamic inventory as follows.

    $ mkdir $HOME/dynamic-inventory
$ cd $HOME/dynamic-inventory
$ cat > openstack.yaml <<EOF
---
plugin: openstack.cloud.openstack
expand_hostvars: yes
groups:
  osp_instances: "'hosts' in openstack.metadata.AnsibleGroup" (1)
compose:
  ansible_ssh_host: openstack.private_v4 (2)
EOF
    Callouts
    1Custom group osp_instances.
    2Setting variable with host private ip address.

3.3. Configure Settings

  1. Let’s setup ansible.cfg in the project directory-

    $ cat > ansible.cfg <<EOF
[defaults]
inventory = openstack.yaml
EOF

  2. Let’s do Ansible Navigator settings, While creating settings file you also need to make sure /etc/openstack/clouds.yaml file mount parmeters are set. As follows.

    $ cat > ansible-navigator.yaml <<EOF
ansible-navigator:
  ansible:
    config: ansible.cfg
    playbook: playbook.yml
  app: run
  execution-environment:
    image: ee-osp:1.1.0
    pull-policy: missing
    volume-mounts:
      - src: "/etc/openstack/clouds.yaml"
        dest: "/etc/openstack/clouds.yaml"
        label: "Z"
  mode: stdout
  playbook-artifact:
    enable: False
EOF

3.4. Verify Inventory

  1. You can verify dynamic inventory entries as follows-

    $ ansible-navigator inventory -i openstack.yaml -m interactive
    Example 1. Steps to list hosts

    1. Type the 1 to Browse hosts.

    2. You will get list of openstack hosts.

    3. To exit, keep on pressing <ESC> key until you are back to shell prompt.

3.5. Run Playbook

  1. Let’s create a simple playbook to test dynamic-inventory as follows -

    $ cat > playbook.yml <<EOF
---
- name: Verify OpenStack Instances
  hosts: osp_instances
  tasks:
    - name: Verify connectivity
      ping:
      register: output
    - debug:
        msg: "{{ output }}"
EOF

  2. You can run playbook against dynamic inventory as follows-

    $ ansible-navigator
    Playbook output
