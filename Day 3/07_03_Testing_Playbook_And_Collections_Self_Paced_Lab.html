1. Start Lab Environment

You need to run following steps if your lab environment has shut down. To conserve resources, the lab environment shuts down automatically after eight hours.

1.1. Start Environment After Shut Down

  1. Go to the OPENTLC lab portal and use your OPENTLC credentials to log in.

  2. Navigate to Services → My Services (this should be the screen shown right after logging in).

  3. In the list of your services, select your lab environment.

  4. Select App Control → Start to start your lab environment.

  5. Select Yes at the Are you sure? prompt.

  6. On the bottom right, click Submit.

    After a few minutes, expect to receive an email letting you know that the lab environment has been started.

1.2. Connect bastion Node

  1. Bastion node is the main host which is accssible via ssh from public network. You will be using this host for running commands and accessing other host available in your environment. You will use ssh to connect bastion node, using the login details provided in third email.

    [laptop]$ ssh <OPENTLC-USER-ID>@bastion.${GUID}.${SUBDOMAIN}

  2. Switch to devops user. The User devops is service account in bastion node who has complete sudoer privilege to run all commands with no sudo password.

    [bastion]$ sudo su - devops

1.3. Refresh Satellite Manifest

  1. After restart of your classroom environment need to run following Lab Manager command to refresh satellite manifest.

    $ lab refresh -i satellite

4. Prepare Scenario

In this part of the lab, you will initialize and configure molecule default scenario in the project directory to test playbook.

4.1. Add Molecule Scenario

  1. You have already cloned molecule-project in previous lab so just change your current directory to project directory as follows.

    (molecule)[node]$ cd $HOME/molecule-project/

  2. To initialize default molecule scenario in the project directory, you can run following command.

    (molecule)[node]$ molecule init scenario -d openstack

4.2. Update molecule.yml

  1. You need to update platforms dictionary in molecule.yml file so it can provision frontend, app1, app2, appdb OpenStack nodes as follows.

    Replace ${guid} with four digit unique string.
    (molecule)[node]$ cat > molecule/default/molecule.yml <<EOF
---
dependency:
  name: galaxy
driver:
  name: openstack                     (1)
platforms:
  - name: frontend                    (2)
    image: rhel-8.3                   (3)
    flavor: GLOBAL-CPU_4_Memory_8192_Disk_30_flavor (4)
    network: ${GUID}-default-network  (5)
    ssh_user: cloud-user              (6)
    groups:
      - loadbalancers                 (7)
  - name: app1
    image: rhel-8.3
    flavor: GLOBAL-CPU_4_Memory_8192_Disk_30_flavor
    network: ${GUID}-default-network
    ssh_user: cloud-user
    groups:
      - backend_servers
  - name: app2
    image: rhel-8.3
    flavor: GLOBAL-CPU_4_Memory_8192_Disk_30_flavor
    network: ${GUID}-default-network
    ssh_user: cloud-user
    groups:
      - backend_servers
  - name: appdb
    image: rhel-8.3
    flavor: GLOBAL-CPU_4_Memory_8192_Disk_30_flavor
    network: ${GUID}-default-network
    ssh_user: cloud-user
    groups:
      - database_servers
provisioner:
  name: ansible
  env:
    ANSIBLE_CONFIG: /home/cloud-user/molecule-project/ansible.cfg (8)
verifier:
  name: ansible
EOF
    Callouts

    6Ssh user for instances.

4.3. Update prepare.yml

  1. You need to update prepare.yml file with host subscription tasks as follows.

    (molecule)[node]$ cat > molecule/default/prepare.yml  <<EOF
---
- name: Prepare
  hosts: all
  gather_facts: false
  become: true
  vars:
    satellite_ca_cert_url: http://satellite.example.com/pub/katello-ca-consumer-latest.noarch.rpm
    satellite_activationkey: gpte-labs-rhel8
    satellite_org: prod
  tasks:
    - name: install katello-ca-consumer package
      ansible.builtin.dnf:
        name: "{{ satellite_ca_cert_url }}"
        state: present
        disable_gpg_check: true

    - name: register system and attach subs
      community.general.redhat_subscription:
        state: present
        activationkey: "{{ satellite_activationkey }}"
        org_id: "{{ satellite_org }}"
EOF

4.4. Update converge.yml`

  1. Now, you can import mycollections.threetierapp.config.yml collection playbook which you need to test as per lab requirements.

    (molecule)[node]$ cat > molecule/default/converge.yml <<EOF
---
- name: Converge
  import_playbook: mycollections.threetierapp.config
EOF

4.5. Update verify.yml

  1. The default molecule scenario does not have useful tasks in molecule/default/verify.yml file to verify the state of the instances. So you can update verify.yml as follows.

    (molecule)[node]$ cat > molecule/default/verify.yml <<EOF
---
- name: Verify
  hosts: loadbalancers
  gather_facts: false
  tasks:
  - name: Access backend server
    uri:
      url: http://localhost:80
      return_content: true
    register: web_output

  - name: Example assertion
    assert:
      that: '"Enter User Name" in web_output.content'
EOF

5. Testing Scenario

In this section, you will run molecule to provision OpenStack instances, execute playbook against the instances and you will also verify instances state as per the molecule default scenario settings and configurations which you did in earlier steps.

  1. Your playbook uses ansible modules from following collection. So you need to install them as follows.

    (molecule)[node]$ cat > $HOME/molecule-project/requirements.yml <<EOF
collections:
  - containers.podman
  - ansible.posix
  - community.general
  - openstack.cloud
  - community.mysql
EOF
(molecule)[node]$ ansible-galaxy collection install -r $HOME/molecule-project/requirements.yml

5.1. Create Sequence

  1. You can run molecule create sequence to provision and prepare OpenStack instances as follows.

    (molecule)[node]$ molecule create
    Create sequence output

5.2. Converge Sequence

  1. You can run molecule converge sequence to execute playbook as follows.

    (molecule)[node]$ molecule converge
    Converge Output

5.3. Verify Sequence

  1. You run molecule verify sequence to check the state of the OpenStack instances after your playbook has finished executing. As follows.

    (molecule)[node]$ molecule verify
    Verify Output

5.4. Destroy Sequence

  1. You can run molecule destroy sequence to deletes all of the provisioned OpenStack instances as follows.

    (molecule)[node]$ molecule destroy

5.5. Test sequence

  1. After running each molecule sequence one by one. Now you can run full lifecycle test sequence as follows.

    (molecule)[node]$ molecule test
    Molecule full test sequence destroys the instance at the end. However we can pass --destroy=never option so molecule will not destroy instance and we can use this instance to test failed task if any. The directory in which the molecule.yml resides is the Scenario’s directory.
