1. Start Lab Environment

You need to run following steps if your lab environment has shut down. To conserve resources, the lab environment shuts down automatically after eight hours.

1.1. Start Environment After Shut Down

  1. Go to the OPENTLC lab portal and use your OPENTLC credentials to log in.

  2. Navigate to Services → My Services (this should be the screen shown right after logging in).

  3. In the list of your services, select your lab environment.

  4. Select App Control → Start to start your lab environment.

  5. Select Yes at the Are you sure? prompt.

  6. On the bottom right, click Submit.

    After a few minutes, expect to receive an email letting you know that the lab environment has been started.

1.2. Connect bastion Node

  1. Bastion node is the main host which is accssible via ssh from public network. You will be using this host for running commands and accessing other host available in your environment. You will use ssh to connect bastion node, using the login details provided in third email.

    [laptop]$ ssh <OPENTLC-USER-ID>@bastion.${GUID}.${SUBDOMAIN}

  2. Switch to devops user. The User devops is service account in bastion node who has complete sudoer privilege to run all commands with no sudo password.

    [bastion]$ sudo su - devops

1.3. Refresh Satellite Manifest

  1. After restart of your classroom environment need to run following Lab Manager command to refresh satellite manifest.

    $ lab refresh -i satellite

4. Hints for Solutions

4.1. Create Project

  1. Ceate $HOME/OpenStack-instances directory.

    Project Solutions

    1. You can use project directory as follows.

    $ mkdir $HOME/OpenStack-instances
$ cd $HOME/OpenStack-instances

4.2. Create Playbook

  1. Create provision.yml playbook and use openstack.cloud.server module to provision instances. FYI, openstack.cloud collection is already inside the ee-osp:1.2.0 Execution Environment.

    Playbook Solutions

    1. Create variables file for instances as per the requirements, as follows-

      $ cat > instances_vars.yml <<EOF
---
instances:
  - name: server1
    keypair: "{{ guid }}-keypair"
    image: rhel-8.3
    flavor: GLOBAL-CPU_4_Memory_8192_Disk_30_flavor
    security_group: "{{ guid }}-HostSG"
    internal_network: "{{ guid }}-default-network"
    metadata:
      AnsibleGroup: osp_instances
      deployment_name: dev

  - name: server2
    keypair: "{{ guid }}-keypair"
    image: rhel-8.3
    flavor: GLOBAL-CPU_4_Memory_8192_Disk_30_flavor
    security_group: "{{ guid }}-HostSG"
    internal_network: "{{ guid }}-default-network"
    metadata:
      AnsibleGroup: osp_instances
      deployment_name: dev
EOF

    2. You can create provision.yml playbook as follows-

      $ cat > provision.yml <<EOF
---
- name: Create OpenStack Instances
  hosts: localhost
  connection: local
  gather_facts: false
  vars:
    guid: "{{ lookup('env','GUID') }}"
  tasks:
    - name: Include Instance variables
      include_vars: instances_vars.yml

    - name: Create Instance
      openstack.cloud.server:
        cloud: "{{ guid }}-project"
        name: "{{ instance.name }}"
        image: "{{ instance.image }}"
        key_name: "{{ instance.keypair }}"
        flavor: "{{ instance.flavor }}"
        security_groups: "{{ instance.security_group }}"
        auto_ip: no
        wait: yes
        nics:
          - net-name: "{{ instance.internal_network }}"
        meta: "AnsibleGroup={{instance.metadata.AnsibleGroup}},deployment_name={{instance.metadata.deployment_name}}"
        state: present
      loop: "{{ instances }}"
      loop_control:
        loop_var: instance
EOF

4.3. Configure Settings

  1. Create Ansible Navigator settings file with default name ansible-navigator.yaml and add all necessary parameters.

    Settings Solutions

    1. You can configure ansible navigator settings file as follows.

      $ cat > ansible-navigator.yaml <<EOF
ansible-navigator:
  ansible:
    playbook: provision.yml
  app: run
  execution-environment:
    image: ee-osp:1.2.0
    pull-policy: missing
    volume-mounts:
      - src: "/etc/openstack/clouds.yaml"
        dest: "/etc/openstack/clouds.yaml"
        label: "Z"
    environment-variables:
      pass:
        - GUID
  mode: stdout
  logging:
    append: False
    level: critical
    file: /tmp/OpenStack-instances/ansible-navigator.log
  playbook-artifact:
    enable: True
    replay: /tmp/OpenStack-instances/artifact.json
    save-as: /tmp/OpenStack-instances/artifact.json
EOF

  1. Execute openstack command line to get list of instances from OpenStack cloud and Make sure you have server1 and server2 in the list.
